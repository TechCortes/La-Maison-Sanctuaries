# Contributing

Thanks for working on **La Maison Sanctuaries**. This project is
developed primarily inside [Lovable](https://lovable.dev), with a
bidirectional GitHub sync. You can contribute from either side.

---

## Workflows

### A) Edit in Lovable (recommended)
1. Open the project in Lovable.
2. Describe the change in chat — Lovable applies it and pushes to GitHub.
3. The deploy preview updates automatically.

### B) Edit locally / on GitHub
1. Clone the repo.
2. Install dependencies: `npm install` (or `bun install`).
3. Run the dev server: `npm run dev`.
4. Commit and push — changes sync back into Lovable in real time.

> Do not edit both sides simultaneously on the same files; let one push
> finish before starting the other.

---

## Code standards

- **TypeScript strict** — no `any` unless justified by a comment.
- **Semantic tokens only** — never hard‑code colors in components.
  Add a token to `src/styles.css` if one is missing.
- **Routing** — file‑based under `src/routes/`. Use flat dot
  filenames (e.g. `settings.profile.tsx`), never nested folders.
- **Imports** — use `@tanstack/react-router` (never `react-router-dom`).
- **Components** — keep them small and focused. Reusable primitives
  belong in `src/components/ui/`; brand‑specific composites at the
  top of `src/components/`.
- **Lint & format** before pushing:
  ```bash
  npm run lint
  npm run format
  ```

---

## Commit hygiene

- One logical change per commit.
- Subject in imperative mood: *"Add investor highlights card"*.
- Reference the route or component touched when possible.

---

## Do not edit

- `src/routeTree.gen.ts` — auto‑generated by the TanStack Router plugin.
- `bun.lockb`, `package-lock.json` — managed by the package manager.

---

## Reporting issues

Open a GitHub issue with:
- A short description of the problem,
- Steps to reproduce (URL, viewport size if visual),
- A screenshot or short screen recording for UI bugs.
